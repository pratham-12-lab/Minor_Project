# 🚀 Quick Render Deployment Guide

## Prerequisites
1. GitHub account with your code pushed
2. MongoDB Atlas cluster ready
3. Cloudinary account configured
4. Environment variables prepared

## Step 1: Deploy with render.yaml (Blueprint)

### Method A: One-Click Deploy (Recommended)

1. **Push to GitHub:**
```bash
git add .
git commit -m "Deploy to Render"
git push origin main
```

2. **Create Render Account:**
- Go to [render.com](https://render.com)
- Sign up with GitHub
- Connect your repository

3. **Deploy with Blueprint:**
- Click "New" → "Blueprint"
- Select your GitHub repository
- Render will read `render.yaml` and create both services automatically

4. **Set Environment Variables:**
After blueprint deployment, add these in Render dashboard:

**Backend Service Environment:**
```
MONGO_URI=your_mongodb_atlas_connection_string
CLOUDINARY_CLOUD_NAME=your_cloudinary_name
CLOUDINARY_API_KEY=your_cloudinary_key
CLOUDINARY_API_SECRET=your_cloudinary_secret
GEMINI_API_KEY=your_gemini_api_key
```

**Frontend Service Environment:**
```
VITE_API_URL=https://your-backend-service.onrender.com
```

## Step 2: Manual Service Creation (Alternative)

### Backend Service:
```
Name: job-portal-backend
Environment: Node
Root Directory: backend
Build Command: npm ci
Start Command: npm start
Auto-Deploy: Yes
```

### Frontend Service:
```
Name: job-portal-frontend
Environment: Static Site
Root Directory: frontend
Build Command: npm ci && npm run build
Publish Directory: dist
Auto-Deploy: Yes
```

## Step 3: Verify Deployment

1. **Check Services:**
   - Backend: `https://your-backend.onrender.com/`
   - Frontend: `https://your-frontend.onrender.com/`

2. **Test API Endpoints:**
   - Health check: `/`
   - Security check: `/api/security/check`

3. **Test Frontend Features:**
   - User registration/login
   - Job posting/browsing
   - File uploads
   - Real-time chat

## Common Issues & Solutions

### Build Failures:
```bash
# If build fails, check:
1. Node.js version (should be 18+)
2. Package.json scripts are correct
3. All dependencies are listed
4. No missing environment variables
```

### CORS Issues:
```bash
# Update FRONTEND_URL in backend environment:
FRONTEND_URL=https://your-actual-frontend-url.onrender.com
```

### Database Connection:
```bash
# Whitelist Render IPs in MongoDB Atlas:
1. Go to Network Access
2. Add IP: 0.0.0.0/0 (Allow all)
3. Or use Render's specific IPs if provided
```

## Environment Variables Reference

### Required for Backend:
- `MONGO_URI` - MongoDB Atlas connection string
- `JWT_SECRET` - Auto-generated by Render
- `CLOUDINARY_CLOUD_NAME` - Your Cloudinary cloud name
- `CLOUDINARY_API_KEY` - Your Cloudinary API key
- `CLOUDINARY_API_SECRET` - Your Cloudinary API secret
- `GEMINI_API_KEY` - Google Gemini API key
- `FRONTEND_URL` - Auto-populated from frontend service

### Required for Frontend:
- `VITE_API_URL` - Auto-populated from backend service

### Optional:
- `SMTP_USER` - Email service username
- `SMTP_PASS` - Email service password
- `OPENAI_API_KEY` - OpenAI API key (if using OpenAI instead of Gemini)
